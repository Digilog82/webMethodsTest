Basic Info > Delete Local File Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Basic Info > Delete Local File
ESB 서버 Local 파일 시스템에 쌓이는 파일의 주기적인 삭제를 위한 정보를 [등록/수정/삭제/조회] 하는 메뉴이다.
ESB 서버 Local 파일 시스템에는 Server Log, Socket Log 등 파일이 계속 늘어나서 파일 시스템의 사용 가능한 공간이 줄어들게 된다.
일정기간 동안만 파일을 저장하고 일정기간이 지나면 파일 시스템에서 파일을 삭제하는 기능을 제공한다.
  • Basic Info > Delete Local File 클릭 : 목록을 조회한다.
  • Create New Local File Delete Path 클릭 : 신규정보 등록화면으로 이동한다.
  • Edit 아이콘 클릭 : 상세조회 및 수정화면으로 이동한다.
  • Delete 아이콘 클릭 : 등록정보를 삭제한다.
Create And Update Screen
Note:  
  • Local File Path는 기존에 등록되지 않은 Unique 한 값으로 해야 한다.
  • 상대경로가 아닌 Full 절대경로로 설정한다.
  • 파일 삭제처리를 하기 위해서는 Basic Info > Local File Delete Schedule 메뉴에서 파일삭제 Service에 대해서 Schedule을 등록해야 한다.
  • OS가 Windows일 경우에는 forfiles.exe 라는 프로그램이 필요하므로 forfiles.exe 설치여부를 확인해야 한다.
  • 모든 Field에 대해서는 필수로 입력한다.
  • 여기에서 등록할 수 있는 디렉토리 예
    SAG_Directory/logs
    SAG_Directory/replicate/outbound
    SAG_Directory/CUSTOM_CONFIG/socket/cauditlogs
    SAG_Directory/CUSTOM_CONFIG/socket/errorlogs
    SAG_Directory/CUSTOM_CONFIG/socket/sauditlogs
    SAG_Directory/CUSTOM_CONFIG/socket/logs
    SAG_Directory/CUSTOM_CONFIG/socket/online/error
    SAG_Directory/CUSTOM_CONFIG/socket/online/tmphis
    SAG_Directory/CUSTOM_CONFIG/socket/online/trace
    SAG_Directory/CUSTOM_CONFIG/socket/skiplogs
  • 운영서버가 두 대인 경우 한 쪽 운영서버에서 [등록/수정/삭제] 하면 다른 쪽 운영서버에서도 자동으로 [등록/수정/삭제] 된다.
  • Config File : SAG_Directory/CUSTOM_CONFIG/localfile/localFileConfig.xml
Local File Path
파일이 계속해서 쌓이는 디렉토리에 대한 전체 경로
이 디렉토리에 있는 파일 및 sub-directory가 삭제된다.
OS가 Windows일 경우에 디렉토리 구분자로 '\' 를 사용하면 Java Script 오류가 발생하므로 '\' 를 '/' 로 바꿔주는 처리를 자동으로 한다.
Remain Days
파일 시스템에 저장하고 있는 보관일수
Schedule 서비스가 실행되는 시점에서 보관일수가 지난 파일 및 sub-directory가 삭제된다.
Sub-Directory Delete YN
파일 뿐만 아니라 sub-directory도 함께 삭제할지 여부
Shared Directory YN
공유 디렉토리인지 여부
운영서버가 두 대인 경우는 일반적으로 공유 디렉토리가 있다.
Schedule 서비스는 두 서버에서 모두 실행되는데 공유 디렉토리인 경우에는 한 쪽 서버에서만 삭제하도록 되어 있다.
Description
Local File Path에 대한 간략한 설명
Backup YN
파일을 삭제하기 이전에 백업을 할 것인지 여부
백업을 할 경우 원본 파일을 Backup Path로 이동하고 원본 파일은 삭제한다.
Backup Path
백업을 할 경우 원본 파일을 이동시킬 경로
