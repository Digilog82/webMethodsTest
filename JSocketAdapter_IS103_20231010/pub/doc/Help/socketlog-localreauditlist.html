Socket Log > Local Server Resubmit Log Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Socket Log > Local Server Resubmit Log
Local Socket Server Port Number로 송수신 되는 Audit Log 중에서 재실행한 Log를 조회하는 메뉴이다.
  • Socket Log > Local Server Resubmit Log 클릭 : 조회화면으로 이동한다.
  • Search Logs 클릭 : 조회조건에 해당하는 Resubmit Audit Log 목록을 조회한다.
  • Transaction ID 링크 클릭 : Resubmit Audit Log 상세조회 화면으로 이동한다.
Search Screen
Note:  
  • Basic Info > Custom Variable 메뉴의 ESB.SOCKET.AUDITLOG.SAVEYN의 Value가 true인 경우에만 로그파일로 저장된다.
  • 로그파일은 날짜별 폴더 아래에 생성된다. 로그파일 Naming Rule은 아래와 같다.
    Port Number or Socket Name + "$" + Start Time + "$" + End Time + "$" + targetSystemName + "$" + interfaceID + "$" + 전문ID + "$" + transactionID + "$" + serverIP + "$" + resultCode + "$" + resubmit
    Custom Variable "ESB.SOCKET.AUDITLOG.SPATH"에 설정된 디렉토리에 저장되는 파일과 같은 이름이며 조회를 목적으로 File Content 없이 파일만 저장하고 상세 데이터 조회 시에는 "ESB.SOCKET.AUDITLOG.SPATH"에 설정된 디렉토리에 저장된 로그 파일을 조회하는 것이다.
  • 파일 저장 위치
    SAG_Directory/CUSTOM_CONFIG/socket/rauditlogs
    운영서버의 경우 NAS 등 공유 디렉토리가 있는 경우에는 공유 디렉토리를 설정해야 로그가 분산되어 저장되지 않는다.
  • Basic Info > Delete Local File 메뉴에서 위의 디렉토리를 등록해서 보관일수가 지난 파일은 자동으로 삭제되도록 할 수 있다.
  • 조회조건 중에서 Log Date는 반드시 입력해야 한다.
  • Transaction ID 링크를 클릭하면 해당 Transaction에 대한 상세 데이터를 조회하는 화면으로 이동한다.
    상세조회 화면에서는 재실행한 Transaction 및 재실행 이전의 Transaction의 기본정보, Socket 통신 로그, 전문의 Schema 정의, 전문의 각 필드의 데이터를 조회할 수 있다.
  • Target System과의 통신방식이 Socket인 경우에 ESB가 Client가 되는데 이런 인터페이스에 대해서도 이 메뉴에서 로그를 조회한다.
