IS Memory Object > Temp Data Delete Schedule Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > IS Memory Object > Temp Data Delete Schedule
Integration Server Memory에 저장하여 관리하는 Object 중에서 보관일수가 지난 데이터를 삭제하는 Schedule을 등록, 관리하는 메뉴이다.
  • IS Memory Object > Temp Data Delete Schedule 클릭 : 목록을 조회한다.
  • Create Schedule 버튼 클릭 : Schedule을 등록한다. Schedule 등록을 하지 않은 상태에서는 Create Schedule 버튼이 Display 된다.
  • Suspend Schedule 버튼 클릭 : Schedule을 중지한다. Schedule을 등록하고 Status가 Active인 경우에는 Suspend Schedule 버튼이 Display 된다.
  • Active Schedule 버튼 클릭 : Schedule을 Active 시킨다. Schedule을 등록하고 Status가 Suspended인 경우에는 Active Schedule 버튼이 Display 된다. Run Time, Remain Days를 수정하고 Active 시킬 수 있다.
Create And Suspend And Active Screen
Note:  
  • Schedule은 하루에 한번 정해진 시간에 실행된다.
  • Schedule 등록 후에 Run Time 및 Remain Days를 수정하고자 할 경우에는 Suppended 상태로 한 다음에 수정하고 다시 Active 상태로 한다.
  • 여기에서 등록한 Schedule은 IS Administrator > Scheduler > Complex Repeating Tasks에서도 확인 가능하다.
  • 아래 메뉴에서 조회되는 데이터들이 삭제 대상이다.
    IS Memory Object > Request Socket Object
    IS Memory Object > Online Request Header
    IS Memory Object > Online Send Status
    IS Memory Object > Online Response Data
    IS Memory Object > Online Reconnecting Socket
    IS Memory Object > Online Receive Sequence
    GW/MA Server가 분리되어 있는 구조에서 Realtime Synch Socket 또는 Asynch Online Socket 전문 송수신 인터페이스가 완료되면 위의 데이터들은 바로 삭제가 되는데 예외상황 발생으로 인해서 삭제되지 않을 수 있다. 삭제되지 않고 남아 있을 경우 JSocketAdapter에서 수행하는 기능에 영향을 주기 때문에 삭제되지 않은 데이터에 대해서 모니터링을 목적으로 일정기간 동안만 저장하고 보관일수가 지나면 삭제처리를 한다.
  • Schedule이 실행되는 일자를 기준으로 해서 데이터가 저장된 날짜가 Remain Days 이전인 데이터가 삭제 대상이 된다.
