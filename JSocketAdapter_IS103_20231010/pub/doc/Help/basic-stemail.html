Basic Info > Source Target Email Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Basic Info > Source Target Email
Interface ID에 대한 Source System 담당자와 Target System 담당자의 Email을 [등록/수정/삭제/조회] 하는 메뉴이다.
  • Basic Info > Source Target Email 클릭 : 목록을 조회한다.
  • Create New Source Target Email 클릭 : 신규정보 등록화면으로 이동한다.
  • Edit 아이콘 클릭 : 상세조회 및 수정화면으로 이동한다.
  • Delete 아이콘 클릭 : 등록정보를 삭제한다.
  • Search 버튼 클릭 : 조회 조건에 해당하는 Interface ID 목록을 조회한다.
Create And Update Screen
Note:  
  • Interface ID는 기존에 등록되지 않은 Unique 한 값으로 해야 한다.
  • 모든 Field에 대해서는 필수로 입력하고 Interface ID를 선택하면 Doc Name, Description은 자동으로 입력된다.
  • 인터페이스 수행 하다가 에러가 발생한 경우 Source System 담당자와 Target System 담당자에게 메일로 에러통보를 할 때 이용된다.
    Basic Info > Custom Variable 메뉴에 등록된 "ESB.CONFIG.ERRORNOTI.STEMAILYN"의 Value가 "true"로 설정되어 있으면서 이 메뉴의 "Email Receive YN" 필드가 "Yes"로 설정되어 있는 경우에만 메일이 전송된다.
  • Deploy Config > Source Target Email 메뉴에서 Deploy 한 이후에 Source System 담당자 또는 Target System 담당자의 메일 정보를 수정한 경우에는 다시 Deploy 가능하도록 deployYN 필드의 값을 "No"로 업데이트 해준다.
  • 운영서버가 두 대 이상인 경우 한 쪽 운영서버에서 [등록/수정/삭제] 하면 다른 쪽 운영서버에서도 자동으로 [등록/수정/삭제] 된다.
  • Config File : SAG_Directory/CUSTOM_CONFIG/socket/sourceTargetEmailConfig.xml
Interface ID
인터페이스 ID
Basic Info > Interface Doc 메뉴에 등록되어 있는 Interface ID와 Basic Info > Business Name > Business Interface List 메뉴에 등록되어 있는 Interface ID를 합쳐서 Display 한다.
Doc Name
전문의 이름. Interface ID를 선택하면 자동으로 셋팅된다.
Description
Interface ID에 대한 간략한 설명. Interface ID를 선택하면 자동으로 셋팅된다.
Email Receive YN
해당 인터페이스 에러 발생 시 메일을 수신할지 여부
Source System Name/Incharge Name/Email
Interface ID에 대한 Source System Name, 시스템 담당자 이름과 Email
Target System Name/Incharge Name/Email
Interface ID에 대한 Target System Name, 시스템 담당자 이름과 Email
