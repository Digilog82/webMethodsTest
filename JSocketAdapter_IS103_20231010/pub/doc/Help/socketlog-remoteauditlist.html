Socket Log > Remote Server Audit Log Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Socket Log > Remote Server Audit Log
Remote Socket Server Port Number로 송수신 되는 Audit Log를 조회하는 메뉴이다.
  • Socket Log > Remote Server Audit Log 클릭 : 조회화면으로 이동한다.
  • Search Logs 클릭 : 조회조건에 해당하는 Audit Log 목록을 조회한다.
  • Transaction ID 링크 클릭 : Audit Log 상세조회 화면으로 이동한다.
Search Screen
Note:  
  • Basic Info > Custom Variable 메뉴의 ESB.SOCKET.AUDITLOG.SAVEYN의 Value가 true인 경우에만 로그파일로 저장된다.
  • 로그파일 Naming Rule은 아래와 같다.
    Port Number or Socket Name + "$" + Start Time + "$" + End Time + "$" + targetSystemName + "$" + interfaceID + "$" + 전문ID + "$" + transactionID + "$" + serverIP + "$" + resultCode + "$" + resubmit
  • 파일 저장 위치
    Custom Variable "ESB.SOCKET.AUDITLOG.CPATH"에 설정한다.
    Socket Server : SAG_Directory/CUSTOM_CONFIG/socket/cauditlogs
    운영서버의 경우 NAS 등 공유 디렉토리가 있는 경우에는 공유 디렉토리를 설정해야 로그가 분산되어 저장되지 않는다.
    Custom Variable "ESB.SOCKET.AUDITLOG.CSUBPATHTYPE"의 설정에 따라서 서브디렉토리 구조가 달라진다.
    CUR_DATE : "ESB.SOCKET.AUDITLOG.CPATH"에 설정된 디렉토리 하위에 "현재날짜" 서브 디렉토리를 생성해서 저장
    SOCKET_NAME : "ESB.SOCKET.AUDITLOG.CPATH"에 설정된 디렉토리 하위에 "현재날짜/소켓Name" 서브 디렉토리를 생성해서 저장
    DOC_ID : "ESB.SOCKET.AUDITLOG.CPATH"에 설정된 디렉토리 하위에 "현재날짜/소켓Name/인터페이스ID" 서브 디렉토리를 생성해서 저장
  • Basic Info > Delete Local File 메뉴에서 위의 디렉토리를 등록해서 보관일수가 지난 파일은 자동으로 삭제되도록 할 수 있다.
  • "ESB.SOCKET.AUDITLOG.CSUBPATHTYPE"의 설정이 "CUR_DATE"인 경우에는 조회조건 중에서 Log Date는 반드시 입력해야 하고 조회조건 중에서 Socket Name, Target System, Interface ID를 "ALL"로 선택하는 경우에는 조회 건수가 너무 많아서 서버 성능에 영향을 줄 수 있기 때문에 Status를 "Fail"로 선택 해야만 조회 가능하다.
  • "ESB.SOCKET.AUDITLOG.CSUBPATHTYPE"의 설정이 "SOCKET_NAME"인 경우에는 조회조건 중에서 Log Date, Socket Name은 반드시 입력해야 한다.
  • "ESB.SOCKET.AUDITLOG.CSUBPATHTYPE"의 설정이 "DOC_ID"인 경우에는 조회조건 중에서 Log Date, Socket Name, Interface ID는 반드시 입력해야 한다.
  • Transaction ID 링크를 클릭하면 해당 Transaction에 대한 상세 데이터를 조회하는 화면으로 이동한다.
    상세조회 화면에서는 Transaction의 기본정보, Socket 통신 로그, 전문의 Schema 정의, 전문의 각 필드의 데이터를 조회할 수 있고 전문의 Schema 정의, 전문의 각 필드의 데이터 및 전문 원본 String 데이터를 엑셀파일로 다운로드 할 수 있다.
  • Source System과의 인터페이스 방식이 Socket이 아니면서 Target System과의 인터페이스 방식이 Socket인 인터페이스에 대해서 이 메뉴에서 로그를 조회한다.
