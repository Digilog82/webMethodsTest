Utility > Deploy File Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Utility > Deploy File
ESB Server Local 파일시스템에 존재하는 파일을 다른 서버에 Deploy 하는 메뉴이다.
  • Utility > Deploy File 클릭 : Deploy 처리 화면으로 이동한다.
  • Deploy File 클릭 : File Deploy 처리를 한다.
Deploy Screen
Note:  
  • 특정 ESB Server의 Local 파일시스템에 존재하는 파일을 다른 ESB Server에 동기화 하고자 할 경우에 이 기능을 사용할 수 있다.
    예를 들어서, JSocketAdapter Administrator의 dsp 파일을 수정하고 다른 ESB Server에도 동기화 하고자 할 경우에 유용하게 사용할 수 있다.
  • Full File Name 그대로 Target Server에 Overwrite 하기 때문에 디렉토리 구조가 같은 파일에 대해서 Deploy 해야 하고 Integration Server 설치 디렉토리가 서로 다른 경우 그 아래에 있는 파일은 상대경로로 해야 한다.
    예를 들어서,
    Source Server 파일 : /ESB01/IntegrationServer/instances/default/packages/JSocketAdapter/pub/index.dsp
    Target Server 파일 : /ESB02/IntegrationServer/instances/default/packages/JSocketAdapter/pub/index.dsp
    위와 같이 Deploy 하기 위해서는 Full File Name을 상대경로인 packages/JSocketAdapter/pub/index.dsp 로 입력해야 한다.

Deploy 하고자 하는 Target Server를 선택
Basic Info > Custom Variable 메뉴의 ESB.CONFIG.REMOTE.DEPFILE에 설정한 목록을 Display 한다.
다중 선택이 가능하고 선택을 하지 않으면 목록에 있는 모든 서버에 Deploy 할 수 있다.
