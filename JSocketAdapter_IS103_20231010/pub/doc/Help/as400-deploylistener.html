AS/400 > Deploy Listener Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > AS/400 > Deploy Listener
AS/400 > Data Q Listener 메뉴에서 관리하는 Listeners 정보를 다른 서버에 Deploy 하는 메뉴이다.
  • AS/400 > Deploy Listener 클릭 : 목록을 조회한다.
  • Search Listener 클릭 : 조회조건에 해당하는 Listeners를 조회한다.
  • Deploy Listener to Disabled 클릭 : 선택한 Listeners 정보를 Disable 상태로 Target Server에 Deploy 한다.
    Target Server에서 직접 Enable 처리를 해야 한다.
  • Deploy Listener to Enabled 클릭 : 선택한 Listeners 정보를 Enable 상태로 Target Server에 Deploy 한다.
    Target Server에서 직접 Enable 처리를 할 필요가 없다.
  • Reserve to Disabled 클릭 : 선택한 Listeners 정보를 Disable 상태로 Target Server에 Deploy 하는 Schedule을 등록한다.
    Schedule 실행 후 Target Server에서 직접 Enable 처리를 해야 한다.
  • Reserve to Enabled 클릭 : 선택한 Listeners 정보를 Enable 상태로 Target Server에 Deploy 하는 Schedule을 등록한다.
    Schedule 실행 후 Target Server에서 직접 Enable 처리를 할 필요가 없다.

Note:  
  • AS/400 메뉴는 AS400 시스템이 존재하고 Data Queue를 이용해서 ESB와 연계하는 경우에만 Display 된다.
    즉, Basic Info > Custom Variable 메뉴에서 ESB.CONFIG.AS400.DATAQUSEYN의 Value가 true로 설정되어 있는 경우에만 Display 된다.
  • AS/400 메뉴는 IS Administrator > Adapters > AS/400 Adapter에서 제공되지 않는 Data Queue Listener 관리에 대한 편리 기능을 제공한다.
    이런 기능을 사용하기 위해서는 IS Administrator > Adapters > AS/400 Adapter > Data Queue Listeners 메뉴에서 등록한 Listener는 AS/400 > Data Q Listener 메뉴에서도 모두 등록하도록 한다.
  • AS/400 > Deploy Listener 메뉴는 개발서버 또는 UAT서버에서만 Display 된다.
  • 여기에서 Deploy 처리를 하면 Target Server의 IS Administrator > Adapters > AS/400 Adapter > Data Queue Listeners 메뉴에 Listener가 등록되는 것이다.
    그러므로, Target Server에서 Listener Node Name에 해당하는 Flow Service가 이미 생성되어 있어야 한다.
  • Reserve to Disabled, Reserve to Enabled는 바로 Deploy 처리를 하는 것이 아니고 Deploy 예약을 하는 것이다.
    Listener 등록은 Listener Node Service가 이미 생성되어 있어야만 등록이 가능하다. 만약, Source Deploy가 야간에 수행될 경우에 유용하게 사용할 수 있는 기능이다.
    Deploy 예약을 하면 정해진 시간에 Deploy 되도록 Schedule 등록을 하는 것으로서 AS/400 > Deployment Reservation 메뉴에서 [조회/수정/삭제] 할 수 있다.
    이 Schedule은 IS Administrator > Server > Scheduler 메뉴에서도 확인할 수 있다.
    이 Schedule은 한 번만 실행되는 것으로서 실행되고 나면 목록에서 자동으로 삭제된다.
    이 Schedule에 의해서 Deploy 처리를 한 후에 Deploy 결과를 ESB 관리자에게 메일로 통보한다.
  • 메일이 정상적으로 전송 가능하기 위해서는 아래 조건을 만족해야 한다.
    Basic Info > Custom Variable 메뉴에서 아래 Custom Variable을 모두 정확하게 설정해야 한다.
    ESB.CONFIG.INCHARGE.EMAILYN : 반드시 true 로 설정
    ESB.CONFIG.EMAIL.HOST : 메일서버의 IP 또는 Host Name
    ESB.CONFIG.INCHARGE.EMAIL : ESB 관리자 이메일 주소
  • AS/400 > Deploy Listener 클릭 시 Search Type은 기본적으로 "Only No Deploy" 이다.
    즉, Deploy 처리를 하지 않은 정보들만 조회를 한다.
    Search Type을 "ALL" 로 변경해서 Deploy 처리 여부와 관계없이 모든 정보를 조회할 수도 있다.
  • Check Box를 선택하지 않고 Deploy 하면 조회된 모든 정보를 Deploy 한다.
  • 운영서버가 두 대인 경우 한 쪽 운영서버에 Deploy 하면 다른 쪽 운영서버에도 자동으로 Deploy 된다.
  • Deploy 처리가 완료된 정보에 대해서는 deployYN이 "Yes" 로 Update 된다.
Target Server
Deploy 하고자 하는 Target Server
Basic Info > Custom Variable 메뉴의 ESB.CONFIG.REMOTE.DEPCONFIG Variable에 등록된 Remote Server Alias 목록
Remote Server Alias가 여러 개일 경우 다중 선택 가능
Deploy 시 선택
Change Library
Local Server와 Target Server의 Library Name이 서로 다른 경우 여기에 입력하면 Library Name이 변경되어서 Deploy 된다.
Deploy 시 입력
Reservation Time
"Reserve to Disabled" 또는 "Reserve to Enabled" 기능을 사용할 경우 Schedule이 실행될 날짜, 시간을 입력한다.
Deploy 시 입력
Deploy to PROD Server, Enabling Server
운영서버에 Deploy 하는 것인지 여부
운영서버가 두 대인 경우 Listener는 한 쪽 운영서버에서만 Enable 되어 있어야 한다.
"Deploy Listener to Enabled" 또는 "Reserve to Enabled" 기능을 사용해서 운영서버에 Deploy 하는 경우에 어떤 서버에서 Enable 처리를 할지 선택해야 한다.
Deploy 시 선택
Provider System Name
Listener가 이용하는 ESB 인터페이스의 Target System Name
조회 시 선택
Business Name
Listener가 이용하는 ESB 인터페이스의 Business Name
조회 시 선택
Search Type
Deploy 하고자 하는 대상에 대한 조회조건
Only No Deploy : Deploy 처리를 하지 않은 정보들만 조회
ALL : Deploy 처리 여부와 관계없이 모든 정보를 조회
