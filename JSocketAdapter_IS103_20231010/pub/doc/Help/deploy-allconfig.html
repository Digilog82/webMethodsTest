Deploy Config > All Config Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Deploy Config > All Config
Deploy Config 메뉴에서 Deploy 하는 모든 정보 중에서 Deploy 처리를 하지 않은 정보들을 일괄적으로 다른 서버에 Deploy 하는 메뉴이다.
  • Deploy Config > All Config 클릭 : 목록을 조회한다.
  • Deploy All Config 클릭 : 아직 Deploy 하지 않은 모든 정보들을 일괄적으로 Target Server에 Deploy 한다.
Search And Deploy Screen
Note:  
  • Deploy Config 메뉴는 개발서버, UAT서버에서는 Display 되고 운영서버에서는 Display 되지 않는다.
  • Deploy Config > All Config 클릭 시 아직 Deploy 처리를 하지 않은 정보들만 조회를 한다.
    단, Schedule 정보는 이 메뉴에서 Deploy 할 수 없고 Deploy Config > Schedule 메뉴에서만 Deploy 할 수 있다.
  • Deploy 하고자 하는 것을 선택할 수 없고 Deploy 하면 아직 Deploy 처리를 하지 않은 모든 정보를 Deploy 한다.
  • 운영서버가 두 대인 경우 한 쪽 운영서버에 Deploy 하면 다른 쪽 운영서버에도 자동으로 Deploy 된다.
  • Deploy 처리가 완료된 정보에 대해서는 deployYN이 "Yes" 로 Update 된다.
  • Remote Server Socket의 Target Socket Server의 IP, Port는 일반적으로 개발서버/UAT서버/운영서버에 따라서 서로 다르므로 Deploy 처리할 때 Change IP & Port 항목의 Target Server 부분에 해당 서버 환경에 맞는 IP, Port 정보를 입력해서 Deploy 한다.
    변경이 없을 경우에는 기본 값 그대로 두거나 입력하지 않아도 된다.
  • Interface Doc, REST API Protocol에서 URL 정보가 포함되어 있으면서 개발서버/UAT서버/운영서버에 따라서 URL이 서로 다른 경우에는 Deploy 한 후에 Target Server에서 직접 수정을 해야한다.
