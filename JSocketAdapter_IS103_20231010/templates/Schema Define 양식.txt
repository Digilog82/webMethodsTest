인터페이스 방식에 따라서 사용하는 Schema Define 양식 선택

1. 엔디안 변환이 없는 Socket To Socket 방식이면서 Source, Target 포맷이 동일한 경우
   DOC_Define_Sample.csv
   DOC_Define_Sample.xls
   
2. 엔디안 변환이 있는 Socket To Socket 방식이면서 Source, Target 포맷이 동일한 경우
   ==> 엔디안 변환이 있는 경우에는 반드시 아래 양식을 사용해야 한다.
   DOC_Endian_Define_Sample1.csv ==> 전문 수신 시 Endian 변환이 필요한 경우
   DOC_Endian_Define_Sample1.xls ==> 전문 수신 시 Endian 변환이 필요한 경우
   DOC_Endian_Define_Sample2.csv ==> 전문 송신 시 Endian 변환이 필요한 경우
   DOC_Endian_Define_Sample2.xls ==> 전문 송신 시 Endian 변환이 필요한 경우
   
3. 엔디안 변환이 없는 Socket To REST API 방식, REST API To Socket 방식, Socket To Socket 방식이면서 Source, Target 포맷이 서로 다른 경우
   (Socket To Socket 방식이면서 Source, Target 포맷이 동일한 경우에도 아래 양식을 사용해도 된다)
   ==> JSON 변환이 필요한 경우에는 반드시 아래 양식을 사용해야 하고 JSON 변환이 필요 없는 경우에는 1, 3번 양식 모두 사용 가능하다. 그러나, JSON 변환이 필요 없는데 굳이 3번 양식으로 복잡하게 할 필요가 없기 때문에
       일반적으로 JSON 변환이 필요 없는 경우에는 1번 양식을 사용하고 JSON 변환이 필요한 경우에는 3번 양식을 사용한다.
   DOC_JSON_Define_Sample.csv
   DOC_JSON_Define_Sample.xls
   
전문정보 등록 시 convertJson(JSON 변환 여부)을 "No"로 설정하면(일반적으로 1, 2번 양식을 사용하는 경우) 변환처리할 때 아래와 같은 서비스들을 사용한다.
JSocketAdapter.COMMON.UTIL.IDATA:createSchemaDef
JSocketAdapter.UTIL:convertToDocument
JSocketAdapter.UTIL:convertToString

전문정보 등록 시 convertJson(JSON 변환 여부)을 "Yes"로 설정하면(일반적으로 3번 양식을 사용하는 경우) 변환처리할 때 아래와 같은 서비스들을 사용한다.
JSocketAdapter.RESTAPI.UTIL:createSchemaDef
JSocketAdapter.RESTAPI.UTIL:convertToDocument
JSocketAdapter.RESTAPI.UTIL:convertToString
