Socket Log > Remote Online Temp History Log Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Socket Log > Remote Online Temp History Log
Asynch Online Socket Client의 인터페이스 시 임시저장하는 데이터에 대한 이력관리 로그파일을 조회하는 메뉴이다.
  • Socket Log > Remote Online Temp History Log 클릭 : 조회화면으로 이동한다.
  • Search Logs 클릭 : 조회조건에 해당하는 로그파일을 조회한다.
  • Download File 클릭 : 조회조건에 해당하는 로그파일을 Download 한다.
Search Screen
Note:  
  • Asynch Online Socket의 경우 요청전문 송신과 응답전문 수신이 Asynch 방식으로 처리되기 때문에 전문 송수신 처리에 필요한 ESB Header, Response Data, Send Status를 IS Memory 또는 파일 시스템에 저장해서 각각의 처리 Step에서 참조하는데 처리가 완료된 후에는 삭제를 한다.
    임시로 저장한 데이터를 이력관리하기 위하여 파일로 저장할 수 있다.
  • Basic Info > Custom Variable 메뉴의 ESB.SOCKET.ONLINE.TMPSAVEYN의 Value가 true인 경우에만 로그파일로 저장된다.
  • 로그파일은 날짜별, Socket Name별로 생성된다. 로그파일 Naming Rule은 아래와 같다.
    OnlineTempDataHistory_SocketName(SystemName_BusinessName)_yyyyMMdd.log(예 : OnlineTempDataHistory_KFTC_BUSI_20180101.log)
    Aynch Online인 경우에는 Socket Name 중에서 SystemName_BusinessName 까지만 파일명에 포함시킨다.
  • 파일 저장 위치 : SAG_Directory/CUSTOM_CONFIG/socket/online/tmphis
  • Basic Info > Delete Local File 메뉴에서 위의 디렉토리를 등록해서 보관일수가 지난 파일은 자동으로 삭제되도록 할 수 있다.
Download Threshold
로그파일을 조회할 때 파일 사이즈가 너무 큰 경우에는 서버 성능에 영향을 줄 수 있다.
Download Threshold에 설정한 값보다 파일 사이즈가 클 경우에는 로그파일을 조회하지 않고 자동으로 파일을 Download 하도록 되어 있다.
즉, Download Threshold에 설정한 값보다 파일 사이즈가 작을 경우에 로그파일을 조회해서 화면에 Display 한다.
