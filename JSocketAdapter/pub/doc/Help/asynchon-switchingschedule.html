KFTC Online > Switching Schedule Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > KFTC Online > Switching Schedule
KFTC Online > Switch Socket Operation 메뉴에서 처리하는 Enable/Disable/Switching 기능에 대한 Schedule을 [등록/수정/삭제/조회] 하는 메뉴이다.
  • KFTC Online > Switching Schedule 클릭 : 목록을 조회한다.
  • Register New Switching Schedule 클릭 : 신규정보 등록화면으로 이동한다.
  • Edit 아이콘 클릭 : 상세조회 및 수정화면으로 이동한다.
  • Delete 아이콘 클릭 : 등록정보를 삭제한다.
Create And Update Screen
Note:  
  • KFTC는 예를 들어서 설명하기 위한 System Name 이다.
  • Basic Info > Business Name 메뉴에서 Asynch Online Socket YN 필드가 'Yes'인 Business Name이 존재할 경우 Provider System Name 단위로 "ProviderSystemName Online" 이라는 메뉴가 생성된다.
  • KFTC Online > Switch Socket Operation 메뉴에서 처리하는 기능에 대해서 만약, 시스템 상황에 따라서 야간에 처리해야 하는 경우에 유용하게 사용할 수 있는 기능이다.
  • 이 메뉴에서 관리하는 Schedule은 IS Administrator > Server > Scheduler 메뉴에서도 확인할 수 있다.
    이 Schedule은 한 번만 실행되는 것으로서 실행되고 나면 목록에서 자동으로 삭제된다.
    이 Schedule에 의해서 Enable/Disable/Switching 처리를 한 후에 처리결과를 ESB 관리자에게 메일로 통보한다.
  • 메일이 정상적으로 전송 가능하기 위해서는 아래 조건을 만족해야 한다.
    Basic Info > Custom Variable 메뉴에서 아래 Custom Variable을 모두 정확하게 설정해야 한다.
    ESB.CONFIG.INCHARGE.EMAILYN : 반드시 true 로 설정
    ESB.CONFIG.EMAIL.HOST : 메일서버의 IP 또는 Host Name
    ESB.CONFIG.INCHARGE.EMAIL : ESB 관리자 이메일 주소
Provider System Name
Remote Socket Server가 위치하고 있는 Remote System의 Name
'KFTC'로 자동으로 설정된다.
Business Name
Enable/Disable/Switching 처리를 하고자 하는 Business Name을 선택
Provider System Name에 속해 있는 Asynch Online Socket Business Name만 Display 된다.
Switching Type
Enable/Disable/Switching 처리 중에서 어떤 처리를 할 것인지 선택
"Socket Operation Switching" Type은 운영서버인 경우에만 Display 된다.
Target Node
Schedule이 실행될 서버의 Server Name을 선택
Start Date
Schedule이 실행될 날짜를 입력
Start Time
Schedule이 실행될 시간을 입력
