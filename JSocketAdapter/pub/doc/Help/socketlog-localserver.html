Socket Log > Local Server Socket Log Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Socket Log > Local Server Socket Log
Local Socket Server Port Number로 송수신 되는 데이터에 대한 로그파일을 조회하는 메뉴이다.
  • Socket Log > Local Server Socket Log 클릭 : 조회화면으로 이동한다.
  • Search Logs 클릭 : 조회조건에 해당하는 로그파일을 조회한다.
  • Download File 클릭 : 조회조건에 해당하는 로그파일을 Download 한다.
Search Screen
Note:  
  • Basic Info > Custom Variable 메뉴의 ESB.SOCKET.LOG.SAVEYN의 Value가 true인 경우에만 로그파일로 저장된다.
  • 로그파일은 날짜별, Port Number별로 생성된다. 로그파일 Naming Rule은 아래와 같다.
    Synch Realtime Port : SocketLog_PortNumber_yyyyMMdd.log(예 : SocketLog_7777_20180101.log)
    Aynch Online Port(송수신겸용) : SocketLog_PortNumber_yyyyMMdd.log(예 : SocketLog_7777_20180101.log)
    Aynch Online Port(수신전용, 송신전용 분리) : SocketLog_INPortNumber_OUTPortNumber_yyyyMMdd.log(예 : SocketLog_7777_7778_20180101.log)
  • 파일 저장 위치 : SAG_Directory/CUSTOM_CONFIG/socket/logs
  • Basic Info > Delete Local File 메뉴에서 위의 디렉토리를 등록해서 보관일수가 지난 파일은 자동으로 삭제되도록 할 수 있다.
Download Threshold
로그파일을 조회할 때 파일 사이즈가 너무 큰 경우에는 서버 성능에 영향을 줄 수 있다.
Download Threshold에 설정한 값보다 파일 사이즈가 클 경우에는 로그파일을 조회하지 않고 자동으로 파일을 Download 하도록 되어 있다.
즉, Download Threshold에 설정한 값보다 파일 사이즈가 작을 경우에 로그파일을 조회해서 화면에 Display 한다.
Port Number
조회하고자 하는 Port Number
Synch Realtime Port, Aynch Online Port(송수신겸용)일 경우에는 Port Number 그대로 Display 되고 Aynch Online Port(수신전용, 송신전용 분리)일 경우에는 INPortNumber_OUTPortNumber로 Display 된다.
