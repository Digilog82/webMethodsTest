Deploy Config > REST API Protocol Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Deploy Config > REST API Protocol
Basic Info > REST API Protocol 메뉴에서 관리하는 REST API Protocol 정보를 다른 서버에 Deploy 하는 메뉴이다.
  • Deploy Config > REST API Protocol 클릭 : 목록을 조회한다.
  • Search REST API Protocol 클릭 : Search Type 조회조건에 해당하는 정보를 조회한다.
  • Deploy REST API Protocol 클릭 : 선택한 REST API Protocol 정보를 Target Server에 Deploy 한다.
  • Edit 버튼 클릭 : 선택한 REST API Protocol 정보를 수정 후 Target Server에 Deploy 하는 화면으로 이동한다.
Search And Deploy Screen
Note:  
  • Deploy Config 메뉴는 개발서버, UAT서버에서는 Display 되고 운영서버에서는 Display 되지 않는다.
  • Deploy Config > REST API Protocol 클릭 시 Search Type은 기본적으로 "Only No Deploy" 이다.
    즉, Deploy 처리를 하지 않은 정보들만 조회를 한다.
    Search Type을 "ALL" 로 변경해서 Deploy 처리 여부와 관계없이 모든 정보를 조회할 수도 있다.
  • URL, User, Password 등 정보는 Deploy Target 서버에서는 다른 값을 설정해야 할 수도 있다. 이런 경우에 "Edit" 버튼을 클릭해서 해당 항목의 값을 수정한 다음 "Deploy REST API Protocol" 버튼을 클릭해서 수정된 정보로 Deploy 할 수 있다.
    현재 서버에서 해당 항목의 값이 수정되는 것이 아니고 현재 서버에서는 그대로 값이 유지되면서 Deploy Target 서버에만 수정된 정보로 Deploy 하는 것이다.
    목록 화면에서 Deploy 하면 현재 서버에서의 값들이 그대로 Deploy 되기 때문에 만약에 Deploy Target 서버에서는 다른 값을 설정해야 하는 항목이 있는 경우에는 Deploy 한 후 Deploy Target 서버에서 직접 수정해야 한다.
  • Check Box를 적어도 하나 이상 선택해야 한다.
  • 운영서버가 두 대인 경우 한 쪽 운영서버에 Deploy 하면 다른 쪽 운영서버에도 자동으로 Deploy 된다.
  • Deploy 처리가 완료된 정보에 대해서는 deployYN이 "Yes" 로 Update 된다.

Remote Server Alias가 여러 개일 경우 다중 선택 가능
Processing Type For Duplicates
Deploy 하는 정보가 Target Server에 이미 등록되어 있을 경우에 대한 처리 기준
Not Add : 중복 에러 메세지를 Alert 하고 등록처리 하지 않음
Update : 기존 등록된 정보를 삭제하고 신규 정보를 등록
Search Type
Deploy 하고자 하는 대상에 대한 조회조건
Only No Deploy : Deploy 처리를 하지 않은 정보들만 조회
ALL : Deploy 처리 여부와 관계없이 모든 정보를 조회
