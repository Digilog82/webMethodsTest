AS/400 > Change Data Queue Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > AS/400 > Change Data Queue
AS/400 > Data Q Listener 메뉴에서 관리하는 Listeners의 Data Queue Name을 변경하기 위한 Schedule을 등록하는 메뉴이다.
  • AS/400 > Change Data Queue 클릭 : 목록을 조회한다.
  • Search Listener 클릭 : 조회조건에 해당하는 Listeners를 조회한다.
  • Reserve Change 클릭 : 선택한 Listeners의 Data Queue Name을 변경하는 Schedule을 등록한다.
Search And Schedule Screen
Note:  
  • AS/400 메뉴는 AS400 시스템이 존재하고 Data Queue를 이용해서 ESB와 연계하는 경우에만 Display 된다.
    즉, Basic Info > Custom Variable 메뉴에서 ESB.CONFIG.AS400.DATAQUSEYN의 Value가 true로 설정되어 있는 경우에만 Display 된다.
  • AS/400 메뉴는 IS Administrator > Adapters > AS/400 Adapter에서 제공되지 않는 Data Queue Listener 관리에 대한 편리 기능을 제공한다.
    이런 기능을 사용하기 위해서는 IS Administrator > Adapters > AS/400 Adapter > Data Queue Listeners 메뉴에서 등록한 Listener는 AS/400 > Data Q Listener 메뉴에서도 모두 등록하도록 한다.
  • Target Node가 개발서버, UAT서버일 경우에는 $any로 등록되고 운영서버일 경우에는 $all로 등록된다.
    즉, 운영서버가 두 대일 경우 모든 운영서버에서 Schedule이 실행된다.
  • Schedule이 실행되면 Basic Info > Business Name 메뉴에 있는 해당 Business Name의 Receive Data Q Name을 변경하고 IS Administrator > Adapters > AS/400 Adapter > Data Queue Listeners 메뉴에 있는 해당 Listener의 Data Queue Name을 변경한다.
  • Listener의 Enable 여부는 기존의 상태를 그대로 유지한다. 즉, 기존에 Disable 상태였으면 그대로 Disable 상태로 하고 기존에 Enable 상태였으면 그대로 Enable 상태로 한다.
  • Data Queue Listener의 Send Data Queue, Receive Data Queue를 변경해야 하는 경우 반영 시점이 야간에 수행될 경우에 유용하게 사용할 수 있는 기능이다.
    변경 예약을 하면 정해진 시간에 변경처리 되도록 Schedule 등록을 하는 것으로서 AS/400 > Queue Change Reservation 메뉴에서 [조회/수정/삭제] 할 수 있다.
    이 Schedule은 IS Administrator > Server > Scheduler 메뉴에서도 확인할 수 있다.
    이 Schedule은 한 번만 실행되는 것으로서 실행되고 나면 목록에서 자동으로 삭제된다.
    이 Schedule에 의해서 변경처리를 한 후에 처리결과를 ESB 관리자에게 메일로 통보한다.
  • 메일이 정상적으로 전송 가능하기 위해서는 아래 조건을 만족해야 한다.
    Basic Info > Custom Variable 메뉴에서 아래 Custom Variable을 모두 정확하게 설정해야 한다.
    ESB.CONFIG.INCHARGE.EMAILYN : 반드시 true 로 설정
    ESB.CONFIG.EMAIL.HOST : 메일서버의 IP 또는 Host Name
    ESB.CONFIG.INCHARGE.EMAIL : ESB 관리자 이메일 주소
  • Check Box는 적어도 한 개 이상 선택해야 한다.
