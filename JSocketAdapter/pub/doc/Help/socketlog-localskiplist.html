Socket Log > Local Server Skip Log Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Socket Log > Local Server Skip Log
Local Socket Server Port Number로 송수신 되는 전문 중에서 Skip 처리된 전문의 Log를 조회하는 메뉴이다.
  • Socket Log > Local Server Skip Log 클릭 : 조회화면으로 이동한다.
  • Search Logs 클릭 : 조회조건에 해당하는 Skip Log 목록을 조회한다.
  • Transaction ID 링크 클릭 : Skip Log 상세조회 화면으로 이동한다.
Search Screen
Note:  
  • Basic Info > Custom Variable 메뉴의 ESB.SOCKET.SKIPLOG.SAVEYN의 Value가 true인 경우에만 로그파일로 저장된다.
  • 로그파일은 날짜별 폴더 아래에 생성된다. 로그파일 Naming Rule은 아래와 같다.
    Port Number + "$" + Start Time + "$" + interfaceID + "$" + 전문ID + "$" + transactionID + "$" + serverIP + "$" + Resubmit YN
  • 파일 저장 위치
    Socket Server : SAG_Directory/CUSTOM_CONFIG/socket/skiplogs
    운영서버의 경우 NAS 등 공유 디렉토리가 있는 경우에는 공유 디렉토리를 설정해야 로그가 분산되어 저장되지 않는다.
  • Basic Info > Delete Local File 메뉴에서 위의 디렉토리를 등록해서 보관일수가 지난 파일은 자동으로 삭제되도록 할 수 있다.
  • 조회조건 중에서 Log Date는 반드시 입력해야 한다.
  • Transaction ID 링크를 클릭하면 해당 Transaction에 대한 상세 데이터를 조회하는 화면으로 이동한다.
    상세조회 화면은 Socket Log > Local Server Audit Log 메뉴에서 이동하는 Resubmit 처리를 할 수 있는 상세화면과 같은 것이다. 이 상세조회 화면에서 해당 Transaction에 대해서 Resubmit 처리가 가능하다.
    Resubmit 한 후에는 Socket Log > Local Server Resubmit Log > Details 화면으로 이동해서 처리결과를 조회한다.
  • Resubmit 버튼을 클릭하면 여러 개의 Transaction을 선택해서 일괄적으로 Resubmit 처리를 할 수 있는데 이 경우에는 Audit Log 데이터 수정 없이 기존의 데이터를 그대로 Resubmit 처리하는 것이다.
    Resubmit 한 후에는 Socket Log > Local Server Resubmit Log 화면으로 이동해서 처리결과를 조회한다.
  • Resubmit 처리는 순차처리/비순차처리 상관 없이 해당 Transaction이 실행된 Main Service를 바로 재실행 하는 것이고 기존의 Transaction이 샐행됬던 서버와 상관 없이 현재 Resubmit 처리를 수행하는 서버에서 Resubmit 처리가 실행된다.
