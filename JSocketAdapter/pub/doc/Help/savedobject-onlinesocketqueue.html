IS Memory Object > Online Socket Queue Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > IS Memory Object > Online Socket Queue
Integration Server Memory에 저장하여 관리하는 Object 중에서 Java Queue 정보 목록을 조회하는 메뉴이다.
  • IS Memory Object > Online Socket Queue 클릭 : 목록을 조회한다.
Search Screen
Note:  
  • Asynch Online Socket Client가 존재하거나 Asynch Online Socket Server가 존재하는 경우에만 이 메뉴가 Display 된다.
  • Basic Info > Custom Variable 메뉴에서 ESB.SOCKET.ONLINE.TRANSTYPE의 Value 설정에 따라서 전문 송수신 Asynch 처리방법이 달라지는데 JMS, MES로 설정하면 UM을 사용하고 INV로 설정하면 UM을 사용하지 않고 Java Queue 클래스를 사용하게 된다.
    Java Queue를 사용할 경우 Client는 Asynch Online Business 별로, Server는 Asynch Online Port 별로 Queue를 생성하고 Queue 정보를 IS Memory에 저장한다.
  • Asynch Online Business Name 생성 시, Asynch Online Business Name Enabled 처리 시에 Client용 Queue가 생성되고 Asynch Online Business Name 삭제 시, Asynch Online Business Name Disabled 처리 시에 Client용 Queue가 삭제된다.
    Asynch Online Socket Server Port Open 시에 Server용 Queue가 생성되고 Asynch Online Socket Server Port Close 시에 Server용 Queue가 삭제된다.
  • Queue의 생성은 ESB.SOCKET.ONLINE.TRANSTYPE의 Value와는 무관하다.
    Asynch Online Business인 경우, Asynch Online Socket Server Port인 경우에 Queue를 사용하든 안하든 무조건 Queue를 생성한다.
