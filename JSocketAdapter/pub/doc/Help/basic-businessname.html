Basic Info > Business Name Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > Basic Info > Business Name
Remote Socket Server가 제공하는 Business에 대한 Name(업무명)을 [등록/수정/삭제/조회] 하는 메뉴이다.
  • Basic Info > Business Name 클릭 : 목록을 조회한다.
  • Create New Business Name 클릭 : 신규정보 등록화면으로 이동한다.
  • Edit 아이콘 클릭 : 상세조회 및 수정화면으로 이동한다.
  • Delete 아이콘 클릭 : 등록정보를 삭제한다.
  • Enabled 필드의 링크 클릭 : Business Name의 사용 여부를 수정한다.
Create And Update Screen
Note:  
  • Business Name은 기존에 등록되지 않은 Unique 한 값으로 해야 한다.
  • Business Name은 영문 이니셜(대문자) 등으로 하고 Custom Variable "ESB.CONFIG.BUSINESSNM.LENGTHCHK"가 true인 경우에는 Custom Variable "ESB.CONFIG.BUSINESSNM.LENGTH"에 설정한 고정길이로 하고 Custom Variable "ESB.CONFIG.BUSINESSNM.LENGTHCHK"가 false인 경우에는 길이에 대한 제한을 하지 않는다.
    Business Name은 언더바( _ )를 포함할 수 없다.
  • Receive Data Q Name, Receive Data Q Key Length 필드는 AS400 시스템이 존재하는 경우에만 입력한다.
    그 이외의 모든 Field에 대해서는 필수로 입력한다.
  • Asynch Online Socket YN 필드를 'Yes' 로 하면 Asynch Online Socket Client로서의 업무를 의미하고 왼쪽 메뉴에 "ProviderSystemName Online" 이라는 메뉴가 Display 된다.
    예를 들어서, Provider System Name이 KFTC(금융결제원) 이라면 "KFTC Online" 이라는 메뉴가 Display 된다.
  • Business Name, Provider System Name에 대한 정보는 그대로 두면서 Asynch Online Socket YN의 정보를 수정할 때 해당 Business Name, Provider System Name에 대한 REST API Protocol 정보를 이미 등록한 경우에는 다음과 같이 webMethods Messaging Trigger의 생성 또는 삭제가 이루어진다.
    Asynch Online Socket YN의 정보를 No에서 Yes로 수정하는 경우에는 webMethods Messaging Trigger를 자동으로 생성하고 Yes에서 No로 수정하는 경우에는 webMethods Messaging Trigger를 자동으로 삭제한다.
    Trigger는 Outbound에 대한 응답전문 송신용, Inbound에 대한 요청전문 송신용 두 개를 생성 및 삭제한다.
  • Socket To REST API 또는 REST API To Socket 방식의 인터페이스를 위해서 Business Name을 등록하는 경우에 "Asynch Online Socket YN" 필드는 Socket으로 연계하는 시스템과의 통신을 기준으로 설정한다.
  • Server Socket > Remote Server Socket 메뉴에서 Socket Name 등록 시 사용된다.
  • Business Name에 대한 [등록/수정/삭제] 정보는 Integration Server Memory에도 똑같이 동기화 된다.
    Integration Server Memory에 저장된 정보는 IS Memory Object > Business Name 메뉴에서 확인 가능하다.
  • 운영서버가 두 대인 경우 한 쪽 운영서버에서 [등록/수정/삭제] 하면 다른 쪽 운영서버에서도 자동으로 [등록/수정/삭제] 된다.
  • Config File : SAG_Directory/CUSTOM_CONFIG/socket/businessNameConfig.xml
Enabled
Business Name에 대한 사용 여부
Business Name
기존에 등록되지 않은 Unique 한 Business의 이름
Socket To REST API 방식 또는 REST API To Socket 방식의 인터페이스가 존재하는 경우에는 같은 업무라 하더라도 Socket Admin > Basic Info > REST API Protocol 메뉴에서 사용할 Business Name과 Socket Admin > Server Socket > Remote Server Socket 메뉴에서 사용할 Business Name을 각각 등록해야 한다.
Provider System Name
Remote Socket Server가 위치하고 있는 Remote System의 Name
Asynch Online Socket YN
Asynch Online Socket 방식으로 처리하는 업무인지 여부
Receive Data Q Name
AS400 시스템이 존재하면서 Data Queue를 이용해서 ESB 시스템에 Request 하는 경우 처리결과를 AS400 시스템에 리턴할 때 사용하는 Receive Data Queue의 Name
Basic Info > Custom Variable 메뉴의 "ESB.CONFIG.AS400.DATAQUSEYN" Variable이 "true"로 설정되어 있는 경우에만 Display 된다.
Receive Data Q Key Length
Receive Data Queue가 Keyed Queue인 경우 Key의 길이
Basic Info > Custom Variable 메뉴의 "ESB.CONFIG.AS400.DATAQUSEYN" Variable이 "true"로 설정되어 있는 경우에만 Display 된다.
