Deploy Config > Doc Schema Define
Screen Heading: IS Administrator > Solutions > Socket Admin > Deploy Config > Doc Schema Define
Basic Info > Interface Doc 메뉴에서 관리하는 전문의 Schema Define 정보를 다른 서버에 Deploy 하는 메뉴이다.
  • Deploy Config > Doc Schema Define 클릭 : 목록을 조회한다.
  • Search Doc 클릭 : 조회조건에 해당하는 전문 목록을 조회한다.
  • Deploy Doc Schema 클릭 : 선택한 전문의 Schema 정의 IData를 Target Server에 Deploy 한다.
  • Reserve Deployment 클릭 : 선택한 전문의 Schema 정의 IData를 Target Server에 Deploy 하는 Schedule을 등록한다.
Search And Deploy Screen
Note:  
  • Deploy Config 메뉴는 개발서버, UAT서버에서는 Display 되고 운영서버에서는 Display 되지 않는다.
  • 전문의 Schema Define IData를 Flow Service에서 관리하지 않고 IS Memory, 로컬 파일시스템에서 관리하는 환경이면서 Main Service의 변경은 없고 전문의 Schema Define만 변경된 경우에 이 메뉴에서 전문의 Schema Define 정보를 Deploy 하면 Package Reloading이 없이 Target Server에서 변경 내용이 바로 적용된다.
  • 이미 다른 서버에 Deploy한 전문 중에서 Socket Admin > Basic Info > Interface Doc 메뉴에서 "Schema Edit" 버튼을 클릭하여 Schema Define 정보를 수정한 전문의 경우만 이 메뉴에서 조회 및 Deploy가 가능하다. 여기에서는 전문의 Schema Define 정보만 Deploy 한다.
  • Check Box를 적어도 하나 이상 선택해야 한다.
  • Target Server를 적어도 하나 선택해야 Deploy 가능하다.
  • Reserve Deployment는 바로 Deploy 처리를 하는 것이 아니고 Deploy 예약을 하는 것이다.
    만약, Source Deploy가 야간에 수행될 경우에 유용하게 사용할 수 있는 기능이다.
    Deploy 예약을 하면 정해진 시간에 Deploy 되도록 Schedule 등록을 하는 것으로서 Deploy Config > Doc Schema Reservation 메뉴에서 [조회/수정/삭제] 할 수 있다.
    이 Schedule은 IS Administrator > Server > Scheduler 메뉴에서도 확인할 수 있다.
    이 Schedule은 한 번만 실행되는 것으로서 실행되고 나면 목록에서 자동으로 삭제된다.
    이 Schedule에 의해서 Deploy 처리를 한 후에 Deploy 결과를 ESB 관리자에게 메일로 통보한다.
