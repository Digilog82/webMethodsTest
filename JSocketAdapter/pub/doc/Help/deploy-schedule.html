Deploy Config > Schedule
Screen Heading: IS Administrator > Solutions > Socket Admin > Deploy Config > Schedule
IS Administrator > Server > Scheduler 메뉴에서 관리하는 Schedule 정보를 다른 서버에 Deploy 하는 메뉴이다.
  • Deploy Config > Schedule 클릭 : 목록을 조회한다.
  • Deploy Schedule to Suspended 클릭 : 선택한 Schedule을 Suspended 상태로 Target Server에 Deploy 한다.
  • Deploy Schedule to Active 클릭 : 선택한 Schedule을 Active 상태로 Target Server에 Deploy 한다.
  • Reserve to Suspended 클릭 : 선택한 Schedule을 Suspended 상태로 Target Server에 Deploy 하는 Schedule을 등록한다.
    Schedule 실행 후 Target Server에서 직접 Active 상태로 바꾸는 처리를 해야 한다.
  • Reserve to Active 클릭 : 선택한 Schedule을 Active 상태로 Target Server에 Deploy 하는 Schedule을 등록한다.
    Schedule 실행 후 Target Server에서 직접 Active 상태로 바꾸는 처리를 할 필요가 없다.
Search And Deploy Screen
Note:  
  • Deploy Config 메뉴는 개발서버, UAT서버에서는 Display 되고 운영서버에서는 Display 되지 않는다.
  • Schedule은 제품의 기능으로서 Deploy 여부를 알 수 없기 때문에 무조건 등록된 모든 Schedule 정보가 조회된다.
  • Check Box를 적어도 하나 이상 선택해야 한다.
  • Target Server가 여러 개일 경우 Target Server를 변경해서 선택하면 해당 Target Server에 등록된 Schedule 실행 Target Node 목록이 Target Node 부분에 조회된다.
  • Reserve to Suspended, Reserve to Active는 바로 Deploy 처리를 하는 것이 아니고 Deploy 예약을 하는 것이다.
    Schedule 등록은 Schedule Service가 이미 생성되어 있어야만 등록이 가능하다. 만약, Source Deploy가 야간에 수행될 경우에 유용하게 사용할 수 있는 기능이다.
    Deploy 예약을 하면 정해진 시간에 Deploy 되도록 Schedule 등록을 하는 것으로서 Deploy Config > Deployment Reservation 메뉴에서 [조회/수정/삭제] 할 수 있다.
    이 Schedule은 IS Administrator > Server > Scheduler 메뉴에서도 확인할 수 있다.
    이 Schedule은 한 번만 실행되는 것으로서 실행되고 나면 목록에서 자동으로 삭제된다.
    이 Schedule에 의해서 Deploy 처리를 한 후에 Deploy 결과를 ESB 관리자에게 메일로 통보한다.

Target Node

Reservation Time
"Reserve to Suspended" 또는 "Reserve to Active" 기능을 사용할 경우 Schedule이 실행될 날짜, 시간을 입력한다.
