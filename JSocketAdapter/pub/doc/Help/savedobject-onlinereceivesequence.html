IS Memory Object > Online Receive Sequence Screen
Screen Heading: IS Administrator > Solutions > Socket Admin > IS Memory Object > Online Receive Sequence
Integration Server Memory에 저장하여 관리하는 Object 중에서 Multi Record 수신 순번 정보 목록을 조회하는 메뉴이다.
  • IS Memory Object > Online Receive Sequence 클릭 : 목록을 조회한다.
  • Delete 아이콘 클릭 : Online Receive Sequence 정보를 삭제한다.
Search Screen
Note:  
  • Asynch Online Socket Client가 존재하는 경우에만 이 메뉴가 Display 된다.
  • Asynch Online Business 응답전문의 구조가 Multi Record 형태이면서 Target System에서 전문의 사이즈 제한으로 인해서 한 번에 모든 Record를 보내주지 않고 여러 번 나눠서 보내주는 경우가 있다.
    JSocketAdapter에서는 응답전문의 수신처리를 Concurrent 하게 처리 하도록 구성하는데 응답전문을 여러 번 나눠서 수신할 경우 순서대로 처리하지 않으면 내부 시스템에서 문제가 발생할 수도 있다.
    수신한 순서대로 처리하기 위해서 현재 처리되고 있는 순번에 대한 정보를 IS Memory에 저장해서 처리되는 순서를 결정하는 데에 이용한다.
  • 전문 송수신 인터페이스가 완료되면 Online Receive Sequence 정보는 삭제를 하기 때문에 일반적으로는 조회되는 정보가 없다. 그러나, 전문 송수신 인터페이스가 수행되는 바로 그 순간에는 잠시 조회될 수도 있다.
    전문 송수신 인터페이스가 완료된 후에도 예외상황 발생으로 인해서 Online Receive Sequence 정보가 삭제되지 않을 수 있다. 이런 경우에는 오류 원인을 파악한 후에 ESB 관리자가 직접 삭제를 할 수 있다.
  • IS Memory Object > Online Data Delete Schedule 메뉴에서 Schedule을 등록해서 보관일수가 지난 데이터는 자동으로 삭제되도록 할 수 있다.
