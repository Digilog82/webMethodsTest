  • IS Memory Object > Local Serial Queuing Socket 클릭 : 목록을 조회한다.
Search Screen
Note:  
  • Basic Info > Custom Variable 메뉴에서 ESB.SOCKET.ONLINE.TRANSTYPE의 Value 설정에 따라서 전문 송수신 처리방법이 달라지는데 JMS, MES로 설정하면 UM을 사용하고 INV로 설정하면 UM을 사용하지 않고 Java Queue 클래스를 사용하게 된다.
    INV로 설정되어 있으면서 Local Server Socket의 Serial Processing YN이 Yes인 경우에 Port 별로 Queue를 생성하고 Queue 정보를 IS Memory에 저장한다.
  • ESB.SOCKET.ONLINE.TRANSTYPE의 Value를 INV로 설정한 후 전문 수신 처리가 최초로 수행되는 순간에 Queue 생성 및 Queue에 들어온 데이터를 처리하는 데몬이 실행되고 이 데몬 서비스에 대한 정보를 IS Memory에 저장한다.
    ESB.SOCKET.ONLINE.TRANSTYPE의 Value를 INV가 아닌 다른 값으로 설정하는 순간 데몬 서비스는 종료되고 Queue 정보 및 데몬 서비스 정보는 IS Memory에서 삭제된다.
    ESB.SOCKET.ONLINE.TRANSTYPE의 Value가 INV라 하더라도 Server Port를 Close 하거나 Client가 Connection을 Close 하면 데몬 서비스는 종료되고 Queue 정보 및 데몬 서비스 정보는 IS Memory에서 삭제된다.
